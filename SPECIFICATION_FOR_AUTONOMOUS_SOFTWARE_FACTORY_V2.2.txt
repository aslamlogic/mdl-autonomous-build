0. PURPOSE, SCOPE, AND DETERMINISM
0.1 This document is the complete, self-contained specification standard for deterministic software generation.
0.2 No external document is required to interpret or implement this standard for P1–P12 execution. All governance required for build execution is embedded in §25.
0.3 The SPS will deterministically construct the system IF AND ONLY IF the spec fully determines the system. Any implementation choice not explicitly constrained by this spec + embedded governance SHALL cause deterministic build failure E-SPEC-UNDERDETERMINED.
0.4 Supra-MetaRules v5.6 §1.1 Truth Discipline applies: The system must not fabricate factual content. Any speculative statement must be explicitly marked as speculation. When uncertain, the system must state: “I cannot confirm this.”

2. POSTGRESQL SCHEMA DEFINITIONS (DATABASE-READY, ALIGNED WITH ER MODEL, VALIDATION LAYER, AND LEGAL RULE ENGINE LAYER)
SQL
-- Matter
CREATE TABLE matter (
matter_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
title TEXT NOT NULL,
status VARCHAR(20) NOT NULL DEFAULT 'active' CHECK (status IN ('active', 'archived')),
user_id UUID NOT NULL,
created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
-- Evidence
CREATE TABLE evidence (
evidence_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
matter_id UUID NOT NULL REFERENCES matter(matter_id),
source_id UUID NOT NULL REFERENCES source(source_id),
content TEXT NOT NULL,
timestamp TIMESTAMPTZ NOT NULL DEFAULT NOW(),
created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
-- Finding
CREATE TABLE finding (
finding_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
evidence_id UUID NOT NULL REFERENCES evidence(evidence_id),
legal_rule_id TEXT NOT NULL,
significance TEXT NOT NULL,
created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
-- Legal Artefact
CREATE TABLE legal_artefact (
artefact_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
finding_id UUID NOT NULL REFERENCES finding(finding_id),
document_type VARCHAR(50) NOT NULL,
content_hash TEXT UNIQUE NOT NULL,
validation_status VARCHAR(20) NOT NULL CHECK (validation_status IN ('provisional', 'validated', 'contested')),
created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
-- Source
CREATE TABLE source (
source_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
provider_adapter_id UUID NOT NULL REFERENCES provider_adapter(adapter_id),
type VARCHAR(50) NOT NULL,
url TEXT NOT NULL,
created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
-- Rule (from Legal Rule Engine)
CREATE TABLE legal_rule (
rule_id TEXT PRIMARY KEY,
source_type VARCHAR(50) NOT NULL CHECK (source_type IN ('statute', 'case_law', 'regulation')),
citation TEXT NOT NULL,
jurisdiction VARCHAR(100) NOT NULL,
condition_set JSONB NOT NULL,
conclusion TEXT NOT NULL,
precedence_level INTEGER NOT NULL,
effective_date DATE NOT NULL,
rule_version TEXT NOT NULL,
created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
-- Provider Adapter
CREATE TABLE provider_adapter (
adapter_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
lpsl_definition_id UUID NOT NULL,
provider_name VARCHAR(100) NOT NULL,
base_url TEXT NOT NULL,
created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
-- Iteration (from SST)
CREATE TABLE iteration (
iteration_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
build_run_id UUID NOT NULL,
iteration_no INTEGER NOT NULL,
started_at TIMESTAMPTZ NOT NULL,
ended_at TIMESTAMPTZ,
status VARCHAR(50) NOT NULL,
action_research_mode BOOLEAN NOT NULL DEFAULT FALSE
);
-- Repo Provision
CREATE TABLE repo_provision (
repo_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
build_run_id UUID NOT NULL,
github_repo_url TEXT NOT NULL UNIQUE,
repo_name TEXT NOT NULL UNIQUE,
visibility VARCHAR(10) NOT NULL CHECK (visibility IN ('private', 'public')),
created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
-- Deploy Provision
CREATE TABLE deploy_provision (
deploy_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
repo_id UUID NOT NULL REFERENCES repo_provision(repo_id),
platform VARCHAR(20) NOT NULL CHECK (platform IN ('render')),
service_id TEXT NOT NULL,
live_url TEXT,
status VARCHAR(20) NOT NULL,
created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
-- LPSL Contract
CREATE TABLE lpsl_contract (
lpsl_definition_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
provider_name VARCHAR(100) NOT NULL UNIQUE,
capability_map JSONB NOT NULL,
request_schema JSONB NOT NULL,
response_schema JSONB NOT NULL,
auth_method VARCHAR(20) NOT NULL CHECK (auth_method IN ('api_key', 'oauth2', 'none')),
rate_limit JSONB NOT NULL,
created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

27. EMBEDDED GOVERNANCE
27.1 To satisfy clause 0.2 and make this artefact standalone for P1–P12 execution, the minimal Supra-MetaRules v5.6 clauses required for build are embedded below. Full SMR remains canonical, but these clauses are inlined to eliminate external dependency.
27.1.1 Truth Discipline: The system must not fabricate factual content. Any speculative statement must be explicitly marked as speculation. No hidden inferential leaps are permitted. When uncertain, the system must state: “I cannot confirm this.”
27.1.2 Evidence Requirements: All factual claims must be supported by evidence or logical derivation. Citations must be inserted at the time of writing. All logical chains must be fully visible within the conversation.
27.1.3 Deterministic Mode: This is the default mode for all governance tasks, academic writing, legal analysis, marking, protocol work, and evidence-based reasoning. No probabilistic reasoning is permitted unless Heuristic Mode is explicitly activated.
27.1.4 Completeness Discipline: All reasoning must be articulated in sequential steps. No compression or omission is permitted unless the user authorises heuristic summarisation.
27.1.5 Multi-Stage Reasoning: Multi-stage reasoning must be linear and fully visible. No collapsing or skipping of reasoning stages is permitted.
27.1.6 Versioning Discipline: Any structural or behavioural modification requires a version increment. Every version of this SST must be fully reproduced in the conversation. Partial updates, deltas, or references to earlier versions are prohibited.
27.1.7 Absolute Ban on Cross-Version Referencing: The system may never reference any clause, section, or wording from a previous version of this SST. Every version must be fully self-contained, independent, and complete. Any attempt to reference a previous version constitutes a structural breach requiring immediate halt.
27.1.8 Pre-Output Self-Audit Gate: Before generating output, the system must check: compliance with all Supra-MetaRules, compliance with the invoked domain protocol, absence of unquoted abbreviations, visibility of reasoning, correct reasoning mode (Deterministic or Heuristic), compliance with the Numbered Options Rule, compliance with the Export Integrity Rule when relevant. If any issue is detected, the system must halt and self-correct.
